# Keyword Research Methodology

> Goal: 50â€“150 *traceable* candidates by the end of Phase 3. Every keyword has a documented source. If you can't explain why a keyword is on the list, it doesn't belong on the list.

## Framework: seed â†’ expand â†’ variations â†’ long-tail

### 1. Seeds (5â€“15 terms)

Start with the project's core nouns:

- The product / service name and its category (*"embedded credit," "value-back platform"*).
- The audience as the audience would describe themselves (*"dealership owner," "venue operator"*).
- The primary problem in the audience's own words (*"interchange fees," "payment processing cost"*).
- 1â€“2 USPs from the brief, generalized (*"merchant-owned loyalty"*).

Seeds are not finished keywords. They are search anchors for expansion.

### 2. Expansion (~30â€“60 terms)

For each seed, generate variations:

- **Modifier-prefixed:** `best X`, `top X`, `cheap X`, `enterprise X`.
- **Modifier-suffixed:** `X software`, `X platform`, `X for [audience]`, `X pricing`, `X reviews`.
- **Question forms:** `what is X`, `how does X work`, `is X worth it`.
- **Comparison forms:** `X vs Y`, `alternatives to Y`, `Y competitors`.
- **Geo-modified** (only if multi-region matters): `X Canada`, `X for US merchants`.

Pull the competitor vocabulary baseline from Phase 2 directly into this list.

### 3. Long-tail variations (~30â€“60 terms)

3â€“7 word phrases that combine intent with specificity:

- *"how to reduce credit card processing fees for dealerships"*
- *"best embedded credit platform for sports venues"*
- *"interchange fee comparison Visa Mastercard 2026"*

Long-tail wins:

- Easier to rank (lower competition).
- Higher commercial intent at the BOFU end.
- Cleaner signal of user need.

Long-tail risks:

- Low absolute volume per term.
- Easy to invent terms nobody searches.

Counter-risk: every long-tail candidate must trace to **at least one** of: competitor page, PAA result, forum thread, pytrends rising query, or related-query API output.

### 4. People Also Ask + forum mining

For each of the top 10 seeds:

- Web-search the seed and capture the PAA block. PAA = literal user questions Google has data for.
- Search Reddit for the seed (`site:reddit.com [seed]`). Read the top 5 threads' titles and self-posts. Capture natural-language questions.
- Same for Quora, Stack Exchange (if technical), niche forums.

Forum / PAA queries are pure user-language and often missing from competitor copy. Goldmine for content TOFU.

## Common modifier sets

| Intent | Modifiers (front or back) |
|---|---|
| **Informational** | how to, what is, why, when, guide to, learn, understand, vs (comparative learning), examples of, types of |
| **Commercial** | best, top, top 10, review, comparison, vs, alternatives to, for [use case], for [industry] |
| **Transactional** | buy, price, pricing, cost, get, free, trial, demo, signup, discount, deals |
| **Navigational** | login, sign in, support, contact, [brand name] |

Reuse this table in Phase 3 as the expansion engine.

## Filtering rules

Apply before exiting Phase 3:

| Rule | Action |
|---|---|
| Term contains a competitor's brand name and isn't a comparison ("X vs [competitor]") | **Discard**. |
| Term is internal company jargon with zero competitor coverage and no PAA / forum trace | **Discard**. |
| Term is too generic to drive qualified traffic (e.g., "fintech," "money") | **Discard or note as too-broad seed.** |
| Term targets the wrong geo / language | **Discard or move to a different locale's list.** |
| Term has obvious negative intent (e.g., "[product] complaints" when the project is a vendor) | **Keep as defensive monitoring only**, mark in Notes. Don't optimize *for* it. |
| Term is duplicate of another with trivial variation (singular/plural, stop-word changes) | **Merge** into one row; note variants in `Notes`. |

## Discard threshold (Phase 3 exit gate)

A candidate that meets **both** of these is dropped:

- Appears in **<2 competitors'** observable surface (titles, H1s, H2s from Phase 2), **AND**
- Has no pytrends signal (relative interest <10 in batch) **or** is unverified (in `with-volumes` mode, the Keyword Planner will resolve this â€” defer the cut to Phase 4).

In `no-volumes` mode, run pytrends *before* finalizing the candidate list. In `with-volumes` mode, keep borderline candidates and let real volumes from Keyword Planner do the cut.

## Source traceability

Every keyword on the final list carries one or more sources in `Notes`:

| Source code | Meaning |
|---|---|
| `comp:N` | Appeared in N competitors' pages from Phase 2. |
| `paa` | Appeared in a PAA block during seed search. |
| `forum:reddit` / `forum:quora` / `forum:other` | Pulled from forum mining. |
| `brainstorm` | Generated from the project brief / USPs (lowest-confidence source â€” must combine with another). |
| `expansion:modifier-X` | Generated by applying modifier X to a seed. Needs validation by another source before promotion to High priority. |

A keyword sourced only as `brainstorm` is suspect. Combine sources for confidence.

## Anti-patterns (don't do this)

- **Don't pad the list to look thorough.** 80 well-sourced keywords > 150 noisy ones.
- **Don't include "head terms" that are too broad to ever convert.** *"credit"* is not a keyword target for a credit platform; *"embedded credit for dealerships"* is.
- **Don't classify by guesswork** â€” when intent is ambiguous, web-search the term and look at the SERP. If the top 10 is editorial blogs, it's informational. If it's vendor landing pages, commercial/transactional.
- **Don't ignore branded queries entirely** â€” *"[your brand]"* and *"[your brand] pricing"* matter for branded SERPs once you launch. Capture them, but don't include them in Phase 4 prioritization (volume is meaningless until brand awareness exists).
